“krakenマニュアル画像挿入について

📸 画像を組み込む【超具体・3ステップ】
【STEP 1】画像（スクショ）を用意して、名前をつける
まずは表示させたい画像を用意します。（※AI生成画像でも良いですが、実際の業務マニュアルであれば、現場のシステムの実際のスクリーンショット（Windowsなら Win + Shift + S など）を使うのが100倍わかりやすくて圧倒的におすすめです！！）

用意した画像に、わかりやすい英語の名前をつけます。

例：メアド重複のSTEP1の画像 👉 dup-email-1.png

例：プラン変更のSTEP2の画像 👉 plan-cancel-2.png
(※日本語のファイル名はやめておきましょう！バグの原因になります！)

【STEP 2】Next.js の public フォルダに画像をぶち込む！
ご主人様の開発している Next.js のフォルダ構成を見ると、app（または src）というフォルダと同じ階層に public という名前のフォルダがあるはずです。
ここは「画像などのファイルを置いておく専用の倉庫」です。

👉 先ほど名前をつけた画像ファイルを、この public フォルダの中にそのままドラッグ＆ドロップで入れてください！！
(※フォルダの中に入れればOKです！)

【STEP 3】コードに「画像の名前」を書き込む！
最後に、先ほど作成した app/procedure-wizard/page.tsx の上の方にある MANUAL_DATA の部分に、imgUrl: "/画像の名前.png" を追記します！

▼ 実際の書き換え例（メアド重複対処法の場合）

// ❌ 変更前（文字だけの状態）
{ 
  step: 1, 
  title: "顧客データの検索", 
  content: "対象のメールアドレスでKraken内を検索し、既存アカウントを特定します。", 
  aiImgDesc: "Krakenの画面で検索窓にメールアドレスを入力している。ダークモードのUI。" 
}

// ⭕️ 変更後（画像ファイル名を指定！）
{ 
  step: 1, 
  title: "顧客データの検索", 
  content: "対象のメールアドレスでKraken内を検索し、既存アカウントを特定します。", 
  imgUrl: "/dup-email-1.png" // 👈 これを追記するだけ！！！（最初のスラッシュ / は必須です）
}
